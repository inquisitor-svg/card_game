# 项目文件说明

这个项目是一个德语学习网站，整体分成三部分：

- `backend/`：Java + Spring Boot 后端，负责提供 API、评分、学习时长建议、训练内容。
- `frontend/`：React 前端，负责网页界面、用户输入、调用后端接口。
- `crawler/`：德语单词爬虫，负责按欧标等级爬取词表。
- Docker 配置：负责把前端和后端打包成容器，一键启动。

## 根目录文件

### `README.md`

项目说明文档。

里面包含：

- 项目功能介绍
- 本地启动后端的方法
- 本地启动前端的方法
- Docker 启动方法
- 外部 API 配置说明

以后如果别人拿到这个项目，通常会先看这个文件。

### `.gitignore`

告诉 Git 哪些文件不要提交。

当前主要忽略：

- `backend/target/`：Java 编译产物
- `frontend/node_modules/`：前端依赖包

这些文件通常可以重新生成，不应该放进代码仓库。

### `docker-compose.yml`

Docker 一键启动配置。

它定义了两个服务：

- `backend`：构建并运行 Spring Boot 后端，暴露 `8080` 端口。
- `frontend`：构建并运行 React 前端，使用 Nginx 托管，暴露 `5173` 端口。

运行：

```powershell
docker compose up --build
```

之后访问：

```text
http://localhost:5173
```

### `backend/src/main/resources/application.properties`

Spring Boot 配置文件。

当前配置：

- `server.port=8080`：后端默认运行在 8080 端口。
- `spring.application.name=german-learning`：应用名称。
- `app.external-evaluation-url`：外部评分 API 地址，可通过环境变量传入。
- `app.external-evaluation-key`：外部评分 API Key，可通过环境变量传入。

如果你想默认改成 8081，可以改：

```properties
server.port=8081
```

## 后端入口和配置

### `GermanLearningApplication.java`

Spring Boot 后端启动入口。

运行这个类，整个后端服务就会启动。

命令行运行：

```powershell
mvn spring-boot:run
```

本质上也是启动这个类。

### `WebConfig.java`

后端跨域配置。

因为前端默认运行在：

```text
http://localhost:5173
```

后端默认运行在：

```text
http://localhost:8080
```

浏览器会认为它们是不同来源，所以需要 CORS 配置允许前端访问后端。

这个文件允许：

- `http://localhost:5173`
- `http://127.0.0.1:5173`

访问 `/api/**` 接口。

## 后端 Controller：接口层

Controller 负责接收前端请求，并把请求交给 Service 处理。

### `PlacementController.java`

入门测试接口。

提供两个 API：

```text
GET /api/placement/questions
POST /api/placement/evaluate
```

作用：

- 返回测试题
- 接收用户答案并返回评分结果

如果你想增加“开始测试”“保存测试历史”等接口，可以在这里加。

### `StudyController.java`

学习时长接口。

提供：

```text
POST /api/study-sessions
```

作用：

- 接收本次学习分钟数
- 返回今日学习时长
- 返回本周学习时长
- 返回学习建议

如果以后要把学习记录保存到数据库，这个 Controller 仍然可以保留，主要改 Service 和数据库层。

### `TrainingController.java`

听说读训练和影子跟读接口。

提供：

```text
GET /api/training/modules
POST /api/training/shadowing/evaluate
```

作用：

- 返回训练模块列表
- 对影子跟读 transcript 进行评分

以后如果接入真实语音 API，可以重点改这里对应的 Service。

## 后端 Service：业务逻辑层

Service 是后端真正处理业务逻辑的地方。

### `PlacementService.java`

入门测试评分逻辑。

它做了几件事：

- 内置测试题
- 支持德译汉和汉译德
- 根据用户答案进行宽松评分
- 根据总分判断水平：
  - `A0`
  - `A1`
  - `A2`
  - `B1`
- 返回学习建议

它不是死板比较固定答案，而是结合：

- 可接受含义
- 关键词命中
- 字符相似度
- 文本归一化

如果你想增加测试题，主要改 `seedQuestions()` 方法。

### `TextSimilarityService.java`

文本相似度工具服务。

它负责：

- 去标点
- 转小写
- 处理德语变音：
  - `ä -> ae`
  - `ö -> oe`
  - `ü -> ue`
  - `ß -> ss`
- 计算词语重合度
- 计算字符相似度

`PlacementService` 和 `TrainingService` 都会用到它。

如果以后接入 AI 语义评分，这个类可以作为本地 fallback。

### `StudySessionService.java`

学习时长记录和建议逻辑。

当前它用内存保存学习时长。

注意：内存保存意味着后端重启后数据会消失。

它会返回：

- 今日累计学习分钟数
- 最近 7 天累计学习分钟数
- 学习建议
- 下一步动作建议

如果以后要正式保存用户学习记录，需要加数据库，然后改这个 Service。

### `TrainingService.java`

听说读训练和影子跟读逻辑。

它负责：

- 提供训练模块
- 影子跟读评分
- 判断是否配置了外部评分 API

当前影子跟读评分是本地启发式评分，主要根据 transcript 和目标句子的相似度判断。

如果要接入外部 API，比如语音识别、发音评分、AI 评价，可以从这里改。

## 后端 Model：数据结构

Model 是前后端传输数据时使用的 Java 对象。

### `PlacementQuestion.java`

入门测试题结构。

字段包括：

- `id`：题目 ID
- `direction`：翻译方向，德译汉或汉译德
- `prompt`：题目内容
- `level`：题目等级
- `acceptedMeanings`：可接受答案含义
- `keywords`：关键词

### `PlacementAnswer.java`

用户提交的单题答案。

字段包括：

- `questionId`：题目 ID
- `answer`：用户答案

### `PlacementSubmission.java`

用户提交的整套测试答案。

字段包括：

- `answers`：多个 `PlacementAnswer`

### `QuestionResult.java`

单题评分结果。

字段包括：

- `questionId`：题目 ID
- `score`：单题分数
- `passed`：是否通过
- `feedback`：反馈文字

### `PlacementResult.java`

整套入门测试结果。

字段包括：

- `overallScore`：总分
- `estimatedLevel`：估计水平
- `recommendation`：学习建议
- `results`：每道题的评分结果

### `StudySessionRequest.java`

前端提交学习时长时的请求结构。

字段包括：

- `minutes`：学习分钟数
- `focusAreas`：学习重点，比如听力、口语、阅读

### `StudySessionResponse.java`

后端返回学习时长建议的数据结构。

字段包括：

- `minutesToday`：今日学习分钟数
- `weeklyMinutes`：本周学习分钟数
- `recommendation`：学习建议
- `nextBestAction`：下一步建议

### `TrainingModule.java`

训练模块结构。

字段包括：

- `id`：模块 ID
- `type`：训练类型，听力、口语或阅读
- `title`：标题
- `level`：等级
- `content`：训练内容
- `shadowText`：影子跟读文本
- `tip`：练习提示

### `ShadowingRequest.java`

影子跟读评分请求。

字段包括：

- `targetText`：目标句子
- `transcript`：用户跟读后的转写文本

### `ShadowingResult.java`

影子跟读评分结果。

字段包括：

- `pronunciationScore`：发音接近度
- `rhythmScore`：节奏分
- `feedback`：反馈文字

### `frontend/Dockerfile`

前端 Docker 镜像构建文件。

它分两步：

1. 用 Node 镜像执行 `npm ci` 和 `npm run build`。
2. 用 Nginx 镜像托管构建出来的静态文件。

Docker Compose 启动前端时会用到它。

### `frontend/nginx.conf`

前端容器里的 Nginx 配置。

它负责：

- 托管 React 打包后的静态页面
- 把 `/api/` 请求代理到后端容器

关键配置：

```nginx
location /api/ {
    proxy_pass http://backend:8080/api/;
}
```

这让前端可以直接请求 `/api`，不用关心后端容器的内部地址。

## 爬虫目录：`crawler/`

### `crawler/README.md`

爬虫使用说明文档。

里面包含：

- 爬虫功能介绍
- 配置文件格式
- 本地示例运行方式
- 输出文件说明
- 真实网页配置方式

如果你想知道爬虫怎么运行，先看这个文件。

### `crawler/cefr_word_crawler.py`

德语单词爬虫主程序。

它负责：

- 读取来源配置
- 按 A1、A2、B1、B2、C1、C2 分级爬取
- 从 HTML 文本中提取德语单词
- 自动去重
- 过滤明显不是单词的内容
- 输出每个等级的 JSON 文件
- 输出总 CSV 文件
- 输出爬取摘要

它只使用 Python 标准库，不需要额外安装第三方包。

常用运行方式：

```powershell
python crawler\cefr_word_crawler.py --sources crawler\sources.example.json --output data\vocabulary
```

如果电脑上没有 `python` 命令，可以试：

```powershell
py crawler\cefr_word_crawler.py --sources crawler\sources.example.json --output data\vocabulary
```

### `crawler/sources.example.json`

爬虫来源配置示例。

它按欧标等级组织来源：

```json
{
  "A1": [
    {
      "url": "crawler/examples/sample_a1.html",
      "note": "Local demo A1 vocabulary page"
    }
  ],
  "A2": [],
  "B1": [],
  "B2": [],
  "C1": [],
  "C2": []
}
```

真实使用时，建议复制成：

```text
crawler/sources.json
```

然后把里面的 URL 换成真实词表网页。

### `crawler/examples/sample_a1.html`

本地测试用的 A1 HTML 页面。

它里面放了一些演示单词，例如：

- `hallo`
- `danke`
- `Kaffee`
- `Wasser`
- `lernen`

这个文件的作用是让你不联网也能测试爬虫是否正常。

## 爬虫输出目录：`data/vocabulary/`

这个目录是爬虫运行后生成的，不需要手动创建。

示例输出：

```text
data/vocabulary/A1.json
data/vocabulary/A2.json
data/vocabulary/B1.json
data/vocabulary/B2.json
data/vocabulary/C1.json
data/vocabulary/C2.json
data/vocabulary/all_words.csv
data/vocabulary/crawl_summary.json
```

### `A1.json` 到 `C2.json`

每个等级一个 JSON 文件。

示例：

```json
[
  {
    "word": "Kaffee",
    "level": "A1",
    "source": "crawler/examples/sample_a1.html"
  }
]
```

### `all_words.csv`

所有等级合并后的 CSV 文件。

适合导入 Excel、数据库或后续导入网站后台。

### `crawl_summary.json`

爬虫运行摘要。

它记录：

- 每个等级配置了几个来源
- 每个等级爬到了多少词
- 生成时间
- 使用的是哪个 source 配置文件
